# 5-Stage Pipelined MIPS Processor — Verilog Implementation

## Architecture Overview

```
        ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐
Clock → │  IF  │→  │  ID  │→  │  EX  │→  │ MEM  │→  │  WB  │
        └──────┘   └──────┘   └──────┘   └──────┘   └──────┘
          PC +        Reg        ALU       D-Mem      Reg
          I-Mem      Ctrl        FWD                  Write
                    Sign-Ext    Branch
                               Target
```

### Pipeline Registers
| Register | Connects   | Key Fields Passed |
|----------|------------|-------------------|
| IF/ID    | IF → ID    | PC+4, instruction |
| ID/EX    | ID → EX    | PC+4, rs/rt/rd, rdata1/2, imm, all control signals |
| EX/MEM   | EX → MEM   | ALU result, rdata2, write addr, zero, branch target |
| MEM/WB   | MEM → WB   | ALU result, mem data, write addr, control signals |

## File Structure

```
mips_pipeline/
├── mips_top.v          ← Top-level module, wires all stages together
├── if_stage.v          ← IF: PC, Instruction Memory, IF/ID reg
├── id_stage.v          ← ID: Register File, Control Unit, ID/EX reg
├── ex_stage.v          ← EX: ALU, ALU Ctrl, Forwarding MUXes, EX/MEM reg
├── mem_stage.v         ← MEM: Data Memory, MEM/WB reg
├── hazard_forwarding.v ← Hazard Detection Unit + Forwarding Unit
├── mips_tb.v           ← Testbench with self-checking assertions
└── program.hex         ← Test program (auto-generated by testbench)
```

## Hazard Handling

### 1. Load-Use Data Hazard (Hazard Detection Unit)
```
  lw  $t0, 0($s0)     ← EX stage: memread=1, rt=$t0
  add $t1, $t0, $t2   ← ID stage: rs=$t0  → HAZARD!
```
**Action:**
- PC stalled (pc_write = 0)  
- IF/ID register frozen (ifid_write = 0)  
- NOP bubble inserted into ID/EX (ctrl_flush = 1)  
- One extra cycle inserted automatically

### 2. EX-EX Forwarding (Forwarding Unit)
```
  add $t0, $t1, $t2   ← EX/MEM: regwrite=1, waddr=$t0
  sub $t3, $t0, $t4   ← EX: rs=$t0 → forwardA = 10
```
**Action:** forwardA/B = `10` → MUX selects EX/MEM ALU result

### 3. MEM-EX Forwarding (Forwarding Unit)
```
  add $t0, $t1, $t2   ← MEM/WB: regwrite=1, waddr=$t0
  nop
  sub $t3, $t0, $t4   ← EX: rs=$t0 → forwardA = 01
```
**Action:** forwardA/B = `01` → MUX selects MEM/WB write-back data

## Simulation

### Using Icarus Verilog
```bash
# Compile all modules
iverilog -o mips_sim \
  mips_top.v if_stage.v id_stage.v ex_stage.v \
  mem_stage.v hazard_forwarding.v mips_tb.v

# Run simulation
vvp mips_sim

# View waveforms
gtkwave mips_pipeline.vcd
```

### Using ModelSim / QuestaSim
```tcl
vlog *.v
vsim mips_tb
run -all
```

### Using Vivado (FPGA synthesis)
1. Add all `.v` files to project  
2. Set `mips_top` as the top module  
3. Set `mips_tb` as simulation top  
4. Map `clk`/`reset` to board clock and button

## Key Design Decisions

- **Register file write-through**: Asynchronous read with same-cycle write-through eliminates the WB→ID hazard for the last pipeline stage, reducing required forwarding paths.
- **Branch resolution in EX**: Branch outcome is determined at the end of EX (not ID), requiring a 2-cycle penalty. The IF/ID and ID/EX registers are flushed on branch-taken via `pcsrc`.
- **Forwarding priority**: EX/MEM forwarding takes priority over MEM/WB forwarding in the combinational forwarding MUX (handles WAW/RAW patterns correctly).
- **$zero protection**: Forwarding and write-back units never forward to/from register $0, preserving its hardwired zero semantics.
